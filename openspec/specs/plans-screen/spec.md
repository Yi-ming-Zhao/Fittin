## Requirements
### Requirement: Plan Library Overview
The system MUST provide a plans screen that helps users browse built-in and custom plans, identify the active plan, and create new plans.

#### Scenario: User browses the plan library
- **WHEN** the user opens the plans screen
- **THEN** the screen shows filter controls for plan categories
- **AND** each plan card communicates plan type, summary information, and whether it is currently active.

### Requirement: Plan Detail And Editing Flow
The system MUST provide a coherent flow from plan library to plan detail and template editing.

#### Scenario: User inspects and edits a plan
- **WHEN** the user opens a plan from the library
- **THEN** the detail view shows weekly structure, progression context, and activation controls
- **AND** the user can enter a template editor that supports editing days, weeks, and exercise structure without layout overflow on narrow screens.
