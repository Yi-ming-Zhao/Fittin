## Requirements
### Requirement: PR Dashboard Overview
The system MUST provide a progress screen that summarizes estimated strength performance and recent milestones across key lifts.

#### Scenario: User opens the progress dashboard
- **WHEN** the user opens the progress screen
- **THEN** the screen shows per-lift performance summaries, trend indicators, and recent milestone entries
- **AND** it offers a progression chart that can be viewed by lift.

### Requirement: Exercise Performance Drilldown
The system MUST allow users to inspect a deeper exercise-specific performance view from the progress area.

#### Scenario: User drills into an exercise
- **WHEN** the user opens an exercise detail view from progress
- **THEN** the screen shows current e1RM context, supporting metrics, layered trend visualization, and session history for that exercise.

### Requirement: Broader Trend Insights
The progress area MUST support a trends and analytics view beyond single-lift PR summaries.

#### Scenario: User reviews training insights
- **WHEN** the user opens a trends or insights view from the progress area
- **THEN** the screen shows consistency and load-oriented analytics rather than only recent PR cards.
